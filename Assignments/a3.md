## 5.2.11

Suppose we already have the definitions of $head$, $tail$, $isnil$ on lists, and $plus$ on Church numbers.

The pseudocode for our recursive function should be: $sum = \lambda l. isnil\ l\ 0\ (plus\ (head\ l)\ (sum\ (tail\ l)))$

But in lambda-caculus, we can't refer to the sum function itself. Let's define a function $SumStep$ which receives a function and a list as parameter: 

$$
SumStep=\lambda f.\lambda l. isnil\ l\ \ 0\ \ (plus\ (head\ l)\ (f\ (tail\ l)))
$$

And we expect the function $f$ to be $SumStep$.

By feeding $SumStep$ to $fix$, we have:

$$
sum=fix\ SumStep=fix\ (\lambda f.\lambda l. isnil\ l\ \ 0\ \ (plus\ (head\ l)\ (f\ (tail\ l))))
$$


## 5.3.7
We define the new *bad values* as follows:

$badnat ::= wrong \mid true \mid false \mid \lambda x.t$

$badbool ::= wrong \mid nv \mid \lambda x.t$

$badfn ::= wrong \mid true \mid false \mid nv$

Then the evaluation rules for stuck terms are as follows:

$if \ badbool \ then \ t_1 \ else \ t_2 \rightarrow wrong$

$succ \ badnat \rightarrow wrong$

$pred \ badnat \rightarrow wrong$

$iszero \ badnat \rightarrow wrong$

$badfn \ t_2 \rightarrow wrong$ (E-App-Wrong1)

$v_1 \ wrong \rightarrow wrong$ (E-App-Wrong2)



## 6.3.2

### Definition of De Bruijn Levels

We first define the **Naming Context** $\Gamma$.

Suppose we have a sequence of variables $\Gamma$: $x_0, x_1, \cdots, x_n$, we use the following function $L_\Gamma$ to transform a named term into unnamed term:

- for variable $x$, we have $L_\Gamma(x)=k$, where $k$ is the index of $x$ in $\Gamma$, counting from left to right.
- for abstraction $\lambda x.t_1$, we have $L_\Gamma(\lambda x.t_1)=\lambda. L_{\Gamma,x}(t_1)$, which means when transfering $t_1$, we append $x$ to the naming context.
- for application $t_1 t_2$, we have $L_\Gamma(t_1\ t_2)=L_\Gamma(t_1)\ L_\Gamma(t_2)$

### Proof of Isomorphism

We need to proove that we can interchangably transfer a level-based representation into an index-based representation, and vice versa.

Suppose $n$ represents the number of $\lambda$ binders in the context, then we define the function $I2L_n$ to transfer an index-based representation into a level-based representation:

- $I2L_n(k)=n-1-k$ (counting from inside and outside sum up to n-1)
- $I2L_n(\lambda .t_1)=\lambda.I2L_{n+1}(t_1)$ (going inside an abstraction, we have one more binder)
- $I2L_n(t_1\ t_2)=I2L_n(t_1)\ I2L_n(t_2)$ 

Symmetrically, we define the function $L2I_n$ to transfer a level-based representation into an index-based representation:

- $L2I_n(k)=n-1-k$
- $L2I_n(\lambda .t_1)=\lambda.L2I_{n+1}(t_1)$
- $L2I_n(t_1\ t_2)=L2I_n(t_1)\ L2I_n(t_2)$ 


Then, we have to prove that, for any unamed representation $t$ and any $n$, we have:

- $L2I_n(I2L_n(t))=t$ (if $t$ is an index-based representation)
- $I2L_n(L2I_n(t))=t$ (if $t$ is a level-based representation)

Thanks to the symmetric of the definition, we only need to prove the first property.

Let's prove it by structural induction on $t$:

- if $t$ is a variable $k$, we have $L2I_n(I2L_n(t))=L2I_n(n-1-k)=(n-1-(n-1-k))=k$
- if $t$ is an abstraction $\lambda.t_1$, we have $L2I_n(I2L_n(t))=L2I_n(\lambda.I2L_{n+1}(t_1))=\lambda.L2I_{n+1}(I2L_{n+1}(t_1))$. According to the induction hypothesis, we have $L2I_{n+1}(I2L_{n+1}(t_1))=t_1$. So $\lambda.L2I_{n+1}(I2L_{n+1}(t_1))=\lambda.t_1$
- if $t$ is an application $t_1\ t_2$, we have $L2I_n(I2L_n(t))=L2I_n(I2L_n(t_1)\ I2L_n(t_2))=L2I_n(I2L_n(t_1))\ L2I_n(I2L_n(t_2))$. According to the induction hypothesis, we have $L2I_n(I2L_n(t_1))=t_1$ and $L2I_n(I2L_n(t_2))=t_2$. So $L2I_n(I2L_n(t_1))\ L2I_n(I2L_n(t_2))=t_1\ t_2$
